# Build Roadmap

## Phase 1: Architecture and cleanup
- Establish standardized folder architecture.
- Add index, standards, templates.
- Move old summary-style content into `99_Legacy` after conversion.

## Phase 2: Convert existing summaries
- Convert initial seed summaries into clinic-facing condition and measure pages.
- Add differential diagnosis, assessment priorities, escalation criteria.

## Phase 3: High-frequency condition expansion
- Add draft-ready pages for high-volume MSK conditions.
- Clearly label pages requiring evidence expansion.

## Phase 4: Assessment + outcome measures
- Build region-based assessment workflows.
- Build core outcome measure library with practical interpretation notes.

## Phase 5: Post-op annexes
- Add procedure-specific pages with protocol variance warnings.
- Add progression framework and escalation triggers.

## Phase 6: Evidence surveillance and updates
- Quarterly scan by condition cluster.
- Refresh evidence language and flagged psychometric values.
