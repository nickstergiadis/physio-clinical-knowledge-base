# Condition Page Template (Clinician-Facing)

Use this structure for point-of-care condition pages. Keep language clinical, concise, and action-oriented. Do not write as patient education.

## Quick View

### Definition / Clinical Framing
- One to two bullets that define the working clinical label and scope.
- Clarify what this label includes and excludes.

### Common Presentation
- Typical symptom distribution, aggravating/easing factors, and function limits.
- Include irritability cues that affect first session planning.

### Key Differentials
- Top competing diagnoses to actively rule in/out.
- Include at least one serious or consequential alternative when relevant.

### Exam Priorities
- Highest-yield objective findings for first visit.
- Include what would shift diagnosis or management.

### Red-Flag Screen
- Fast referral/escalation checks and urgent triggers.
- Keep this practical and triage-oriented.

### First-Line Management
- Conservative-first plan and key education points.
- Mention what *not* to over-prioritize when evidence is weak.

### Prognosis / Timeline Snapshot
- Recovery range and expected trajectory checkpoints.
- Frame as range + reassessment, not certainty.

## Deep View

### Differential Reasoning
- Compare pattern fit across likely diagnoses.
- State which findings raise/lower confidence for each.

### High-Yield Tests and Interpretation
- Cluster-based test strategy with interpretation caveats.
- Avoid single-test deterministic language.

### Subgrouping / Modifiers
- Irritability, load tolerance, tissue healing stage, psychosocial context, sport/work demand.
- Specify how modifiers change treatment emphasis.

### Treatment Options
- Tiered options (education/exercise first, adjuncts second, escalation criteria).
- Include referral and imaging criteria where they alter management.

### Exercise / Progression Strategy
- Stage-based progression (acute, subacute, late/RTS where relevant).
- Entry criteria, progression criteria, flare-response rules.

### Follow-Up Markers / Reassessment
- Function markers, symptom behavior, and objective markers to track.
- Include suggested interval and fail-to-progress triggers.

### Evidence Summary
- Synthesize strongest current evidence level and certainty limits.
- Highlight controversies or evidence gaps.

### Linked Related Pages
- Link to relevant special tests, treatments, exercise progressions, outcome measures, and evidence updates.
- Use internal KB labels that resolve to app routes.

## Citations
- Include source links in markdown format: `[Label](https://...)`.
- If citation details are pending, use clearly labeled placeholders (e.g., `PLACEHOLDER CITATION - verify`).
