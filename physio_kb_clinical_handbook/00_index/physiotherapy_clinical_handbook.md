# Physiotherapy Clinical Handbook

## Purpose
This handbook is the front-door guide to the physiotherapy knowledge base. It is designed to help you navigate the full folder set quickly in clinic, keep cluster coverage consistent, and use the package as a reasoning-support tool rather than a static set of notes.

## What is in the package
- cluster-based foundations
- condition pages
- assessment tools
- rehab progression frameworks
- evidence update pages
- post-op protocol annexes where relevant

## How to use the knowledge base in clinic
1. Identify the dominant presentation.
2. Open the condition page that best fits the presentation.
3. Pair it with the relevant assessment tool.
4. Use the progression framework to stage loading and function.
5. Check the evidence update page if your assumption may be outdated.
6. Use post-op annexes only after reconciling them with the actual operation and restrictions.

## Quick-start by presentation
### Low back pain
Start with non_specific_low_back_pain.md, then lumbar_subjective_objective_exam.md, then lumbar_rehab_progression_framework.md. If leg-dominant or below-knee symptoms are present, open lumbar_radicular_pain_sciatica.md early.

### Hip or groin pain
Use hip_subjective_objective_exam.md to separate intra-articular, lateral hip, and referred patterns. Then choose hip_osteoarthritis.md, femoroacetabular_impingement_syndrome.md, or greater_trochanteric_pain_syndrome.md.

### Knee pain
Use knee_subjective_objective_exam.md first. Match to knee_osteoarthritis.md, patellofemoral_pain.md, or degenerative_meniscal_tear.md. If this is post-op ACLR/meniscal repair, switch early to the post-op ACL condition and progression framework.

### Ankle injury
Use ankle_subjective_objective_exam.md, then select lateral_ankle_sprain.md, chronic_ankle_instability.md, or midportion_achilles_tendinopathy.md depending on the pattern.

### Shoulder pain
Use shoulder_subjective_objective_exam.md to separate RCRSP, frozen shoulder, and instability. Then pair the condition page with shoulder_rehab_progression_framework.md.

### Neck pain or headache
Use cervical_subjective_objective_exam.md to determine neck-dominant, headache-dominant, or arm-symptom presentations. Then open mechanical_neck_pain.md or cervicogenic_headache.md.

### Elbow / forearm pain
Start with elbow_forearm_subjective_objective_exam.md. Use tendon pages when pain is local and load-linked; use the nerve entrapment differential add-on if symptoms are burning, numb, weak, or diffuse.

### Hand / wrist pain
Use hand_wrist_subjective_objective_exam.md to map the pain site and task triggers, then choose de_quervain_tenosynovitis.md or hand_wrist_tendinopathy_load_related.md.

### Jaw / TMJ pain
Use tmj_subjective_objective_exam.md, then choose myogenous_temporomandibular_disorder.md or tmj_disc_displacement_mobility_disorder.md.

## Operating principles
- Use the package to support clinical reasoning, not to replace it.
- Do not over-diagnose from a single special test or static postural finding.
- Keep irritability, functional limits, and comparable signs central.
- Use exercise and graded exposure as the long-term backbone of care in most MSK cases.
- Use passive treatment and symptom modifiers as adjuncts, not the whole plan.
- Escalate or refer if the pattern is non-mechanical, progressive, or inconsistent with routine MSK care.

## Post-op annex use rules
- The annexes are practical notes, not surgeon-specific protocols.
- Always check the operative note, tissue quality, concomitant procedures, and restrictions.
- Use milestone logic where appropriate, but do not force generic timelines onto atypical procedures.

## Documentation habits to pair with the KB
- Record dominant classification and irritability.
- Record one or two comparable signs or tasks.
- Document red flags screened and key negatives when relevant.
- Link exercise selection to the dominant impairments and function, not to diagnosis alone.
- Track outcome measures consistently rather than changing tools every visit.

## Maintenance plan
- Review evidence-update pages periodically and refresh cluster assumptions when guidance changes.
- Add new case-driven topics only when they recur or clearly matter to your caseload.
- Keep file names stable once they are in regular use so cross-referencing stays clean.

## Important caveat
This package supports physiotherapy reasoning and rehab planning. It does not replace medical diagnosis, urgent triage, specialist input, or surgeon-specific orders.
